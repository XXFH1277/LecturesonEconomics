# 来源核验规范

## 核验原则

1. 最新内容必须联网核验，不能只依赖记忆。
2. 每个关键事实都要有来源，尤其是价格、财报、基金持仓、政策、库存、战争、制裁和宏观数据。
3. 优先使用原始来源；如果使用媒体报道，要尽量回到公告、文件、统计表或官方声明。
4. 对尚未确认的信息，要明确标注为“待确认”或“市场推测”。
5. 对来源之间的冲突，要说明冲突点和采用哪个来源的原因。

## 来源优先级

### 教材和课程框架

- MIT OpenCourseWare Principles of Microeconomics: https://ocw.mit.edu/
- MIT OpenCourseWare Principles of Macroeconomics: https://ocw.mit.edu/
- OpenStax Principles of Economics: https://openstax.org/details/books/principles-economics-3e
- CORE Econ The Economy: https://www.core-econ.org/the-economy/
- 常见高校经济学、金融学、投资学导论教材可用于安排概念顺序，但不能替代实时数据和原始来源。

教材和公开课主要用于“由简入繁的教学顺序”，例如稀缺性、机会成本、供给需求、市场均衡、货币、通胀、利率和金融市场。现实案例、公司情况、基金数据和市场价格仍必须回到原始或权威来源核验。

### 公司和股票

- 公司 IR、年报、季报、10-K、10-Q、20-F、招股书。

- 审计报告、投资者演示材料、业绩电话会文字稿。

- 指数公司方法论，例如 MSCI、S&P Dow Jones Indices、FTSE Russell、中证指数。

## 每次引用格式

建议使用：

> 数据或事实，来源：机构/文件名，发布日期或数据截至日期，链接。

如果同一段里有推理，必须写清楚：

- 已确认事实：
- 我的推理：
- 后续验证：

## GitHub 使用边界

GitHub 可以用于：

- 寻找数据下载脚本、可视化项目、研究框架、回测工具。
- 学习别人如何组织数据和建模。
- 保存自己的课程笔记、模板和复盘。

GitHub 不可以单独作为真实市场事实的最终来源。任何来自仓库的数据、图表或结论，都必须继续追溯到原始数据机构或官方披露。

## 用户提供资料使用边界

用户提供的 PPT、字幕、文章、视频笔记、网文整理和社群材料可以用于：

- 发现适合零基础学习的讲课顺序。
- 提炼常见问题、常见误区和练习题。
- 作为课堂讨论材料或案例线索。

用户提供资料不可以单独作为市场事实、政策事实、法律规则、历史行情、投资者盈亏比例、公司经营情况或具体买卖结论的最终来源。涉及事实判断时，必须继续追溯到官方、监管、交易所、公司、基金、统计机构或权威研究来源。
